# Contributing to OF-Scraper

Thank you for considering a contribution! We really appreciate the help. To ensure your hard work gets merged quickly and doesn't run into technical hurdles, please keep our branching strategy in mind:

### 🌿 Which branch should I use?

The `main` branch usually lacks the latest code and internal changes

* **Please target `dev/3.XX`:** All new features and bug fixes should be opened against the current development branch.
* **Avoid `main` for PRs:** If you start your work from `main`, you might run into heavy merge conflicts or find that your fix has already been addressed in `dev`. To save yourself the headache of re-doing your work, **please check out `dev/3.XX` before you start.**

### 🚀 Quick Start
1.  **Fetch** the latest `dev/3.XX`.
2.  **Branch** off of it for your fix.
3.  **Submit** your PR back to `dev/3.XX`.
